**分工**：

| skill | 职责 |
| --- | --- |

## 一、实验编号规则（Exp1 专用）

### 1.1 格式

```
{主版本}.{次版本}[t{微调序号}]
```

| 示例 | 含义 |
| --- | --- |
| `7.0` | 7 系列首个大改实验 |
| `7.0t1` | 7.0 基础上的第一次微调 |
| `7.0t2` | 7.0 基础上的第二次微调 |
| `7.1` | 7 系列下一次大改（新奖励、IK 逻辑、URDF 等） |

- **主版本**：阶段号。当前活跃系列为 **7.x**（5.x 为历史，不再新增）
- **次版本**：同系列内大改递增：`7.0 → 7.1 → 7.2`
- **`t` 后缀**：仅权重/单参数/小公式微调，继承上一非 t 实验的修改编号

### 1.2 何时用 `t`（微调）

满足**任意一条**即用 `t`：

- 仅改奖励权重（1~3 项）
- 仅改 `swing_height` / `step_length` / `cycle_time` 等轨迹参数
- 回滚上一轮某个参数
- 训练未达预期，小幅修正后重跑

**示例**：`7.0` 训完对称性仍差，只把 `leg_symmetry` 从 1.0 调到 1.5 → `7.0t1`

### 1.3 何时递增次版本（大改，无 `t`）

- 新增/删除奖励函数
- 修改 IK / `compute_ref_state()` 逻辑
- 修改 URDF、观察空间、网络结构
- 单次改动涉及 **≥2 个独立模块**（如 IK + 新奖励）
- 阶段目标切换（如从「对称」进入「高速」）

**示例**：7.0 解决对称后，下一阶段加速度跟踪专项 → `7.1`

### 1.4 编号不可变

- 失败实验保留编号，索引表状态标「失败」
- 不删除、不合并未发布的实验节
- 不跳号（7.0 之后应是 7.0t1 或 7.1，不要直接 8.0 除非新开系列并说明）

## 二、索引表更新规则

文件顶部 `## 实验索引` 表格，**每新建实验必须先加一行**：

| 列 | 必填 | 说明 |
| --- | --- | --- |
| 编号 | ✅ | 如 `7.0t1` |
| 日期 | ✅ | `YYYY-MM-DD` |
| 摘要 | ✅ | 一句话：改了什么、解决什么 |
| 状态 | ✅ | 见下表 |

**状态枚举**：

| 状态 | 何时使用 |
| --- | --- |
| 待训练 | 方案已写，未开训 |
| 训练中 / 续训中 | 正在跑 |
| 已测试 | 有 CSV，结论已写 |
| 失败 | 训练或回放未达预期，保留教训 |
| 已废弃 | 方向否定，不再续做 |

可选扩展列（推荐 7.1 起加上）：`CSV` | `checkpoint`

## 三、修改内容编号规则

### 3.1 `t` 系列：修改一、修改二…（跨 t 实验继承）

```markdown
## 实验 7.0
### 修改一：…
### 修改二：…

## 实验 7.0t1
### 修改三：…    ← 继承 7.0，继续累加
```

### 3.2 大改（7.0 → 7.1）：实验一、实验二…（重置）

```markdown
## 实验 7.1
### 实验一：…    ← 从「实验一」重新开始
### 实验二：…

## 实验 7.1t1
### 实验三：…    ← 继承 7.1
```

| 场景 | 小节前缀 | 是否继承 |
| --- | --- | --- |
| 7.0（系列首个大改） | 修改 | 修改一 起 |
| 7.0t1（微调） | 修改 | ✅ 继承 7.0 |
| 7.1（新大改） | 实验 | ❌ 重置为实验一 |
| 7.1t1（微调） | 实验 | ✅ 继承 7.1 |

## 五、Agent 工作流

### 5.1 新建实验

1. 判断编号：微调 → `{base}t{n}`；大改 → 递增次版本
2. 在索引表**顶部数据行后**追加一行（新实验在表末）
3. 在文件**末尾**追加完整实验章节（§1~§7，§7 可占位）
4. §1 必须引用上一实验的**具体数值**，禁止只写「效果不好」

### 5.2 训练完成后更新结果

1. 运行分析：

```powershell
$env:PYTHONIOENCODING='utf-8'
python czy/skills/skills/isaac-diag-eval/eval_isaac_diag.py czy/data/isaac_diag_<新>.csv `
  --compare czy/data/isaac_diag_<旧>.csv --compare-label "<上一轮编号>"
```

2. 将 E1/E2/E3、§7 速度跟踪、§8 落地冲击写入 **§7 实验结果** 表
3. 写分级结论（✅改进 / ⚠️部分改进 / ❌未改进）
4. 更新索引表：状态 → `已测试` 或 `失败`；摘要可补关键判定

### 5.3 微调重跑（t 实验）

1. 新编号 `{parent}t{n}`
2. §1 引用 parent 实验 §7 的结论
3. §3 修改编号继承 parent 的「修改/实验」序列
4. §6 验收表的「上一轮」列填 parent 的实测值

### 5.4 禁止事项

- ❌ 不新建 `analyze_5_*.py`（用 isaac-diag-eval）
- ❌ 不在 §7 编造未跑脚本的数据
- ❌ 不删除失败实验记录
- ❌ 不把 E3 pos HF% 与第 3 节 vel HF 占比混为一谈

## 八、与历史 5.x 记录的关系

- **5.1~5.8 不强制回溯改写**；结构可保持现状
- 新实验从 **7.0** 起严格执行本模板
- 引用历史结论时，直接写「实验 5.8」并带 CSV 文件名与关键数值
- 若需对照 5.x 指标，用 isaac-diag-eval `--compare` 拉旧 CSV

## 九、注意事项

1. **§1 教训必须可量化**：至少 2 个来自 isaac_diag 或 TensorBoard 的数
2. **失败同样完整记录 §7**：失败原因写在根因分析
3. **索引表与正文状态一致**
4. **结论分级**：✅改进 / ⚠️部分改进 / ❌未改进，禁止笼统「还行」
5. **大改判断不清时宁可不用 t**：用 `7.1` 而非 `7.0t5`
